# Firmware integration notes

This directory contains HAL/CubeMX-oriented application code for the two
STM32F103C8T6 boards. The files are intentionally kept separate from the host
CMake build because the STM32 HAL drivers are generated by CubeMX or provided by
STM32CubeIDE.

Recommended workflow:

1. Create one CubeMX project for the master board and one for the slave board.
2. Configure the peripherals described in `docs/cubemx_setup.md`.
3. Copy `core/include` and `core/src` into the CubeMX project, or add them to
   the IDE include/source list.
4. Copy `app_entry.c` plus the needed app file from this directory:
   - `app_master_uart.c` and `app_slave_uart.c` for the required UART version.
   - `app_master_can.c` and `app_slave_can.c` for the optional CAN version.
5. Define exactly one role macro (`BOARD_MASTER` or `BOARD_SLAVE`) and exactly
   one link macro (`LINK_UART` or `LINK_CAN`).
6. Call `app_init()` once after CubeMX peripheral init, then call
   `app_task(HAL_GetTick())` in the main loop.

The timer for SG90 PWM should be configured as 50Hz with a 1MHz counter clock,
so writing `CCR = pulse_us` directly produces a 0.5ms to 2.5ms high pulse.
